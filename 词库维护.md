# AiWord 词库维护与更新指南

AiWord 的核心亮点是**“云端大模型与本地极速词库”**的结合。为了让你能够优雅地管理、更新甚至分发你的专属词汇库，请参考以下维护规范与更新策略。

---

## 一、核心原理：JSON 到 BIN 的演化

在 AiWord 的底层设计中：
1. **JSON 是“人类可读”的维护格式**：所有的开源词库、你手动整理的词条、大模型导出的数据，最终都应整理为结构化的 `.json` 格式。
2. **BIN 是“机器读取”的极速缓存**：无论是通过直链下载还是本地导入，当 App 接收到 JSON 后，会在后台将数据序列化（压缩并提取关键索引），生成 `words.bin` 或 `roots.bin`，并隐藏保存在手机系统的沙盒文件系统（DocumentDirectory）中。
3. **日常查词只读 BIN**：生成 `.bin` 缓存后，哪怕你有十万词条，App 的毫秒级本地搜索、划卡特训等功能，读取的都是沙盒里的 BIN 文件，从而做到完全脱机且不占多余内存。

---

## 二、维护与更新词库的三大方案

### 方案 1：GitHub / Gitee 静态托管（⭐ 强烈推荐的 OTA 热更新方案）
**适用场景**：你希望在不更新 App 安装包（APK）的前提下，让用户（或你自己）随时获取最新的词汇数据。
**操作流程**：
1. **维护仓库**：在 GitHub 建一个公开仓库，将你的 `words.json` 和 `roots.json` 放上去，并持续提交 (Commit) 更新。
2. **获取直链**：使用 GitHub Raw 链接，或者借助 `jsDelivr` 获取加速直链（例如：`https://cdn.jsdelivr.net/gh/你的用户名/仓库名@main/words.json`）。
3. **App 内置**：如果想更人性化，可以将默认的直链地址写死在 App 设置页【通过 URL 下载】的默认输入框里。
4. **用户更新**：用户觉得词库旧了，只需到设置页点击“下载更新”，App 就会重新拉取最新 JSON 并在后台重新编译覆写本地的 `.bin`。

### 方案 2：随 App 安装包出厂内置打包（开箱即用方案）
**适用场景**：你希望用户一安装完 App 就能离线查词，不需要任何额外的下载动作。
**操作流程**：
1. 把准备好的 `words.json` / `roots.json`（甚至可以直接放你之前在手机上跑出来的 `words.bin`）放到 React Native 项目的 `android/app/src/main/assets/` 目录下。
2. 在 `offlineDict.ts` 里增加一段初始化代码：利用 `react-native-fs` 的 `RNFS.readFileAssets('words.json', 'utf8')` 或 `RNFS.copyFileAssets()` 方法。
3. 设定逻辑：App 首次启动时，检查沙盒目录是否存在 `.bin`。如果不存在，则静默读取 `assets` 里的文件并生成缓存。之后查词均走缓存。

### 方案 3：本地选取导入（极客/玩家方案）

**适用场景**：高级用户拥有自己专属的、不想公开的生词本。
**操作流程**：
1. 用户在电脑端整理好 JSON。
2. 发送到手机微信或直接丢进手机存储。
3. 打开 AiWord 设置页，点击【从本地选择导入】，App 会调起手机自带的文件管理器。选中后，直接覆写并重新编译个人 `.bin` 词库。

## 三、词库标准数据格式 (JSON Schema)
无论你怎么更新，请确保你维护的 JSON 数组内，每个 Object 遵循 App 定义的字段结构，否则在生成 BIN 缓存时会报错或丢失信息。

### 1. 单词库 (`words.json`) 标准结构示例：
```json
[
  {
    "word": "abandon",
    "phonetic_us": "əˈbændən",
    "translation": "v. 放弃，遗弃；n. 放任",
    "breakdown": [
      { "prefix": "a-", "meaning": "加强语气" },
      { "root": "band", "meaning": "命令、控制" }
    ],
    "context": "general",
    "memory_story": "加强(a)命令(band)，即不顾一切地放弃。"
  }
]
```
*(注：`breakdown`、`phonetic_us` 等非关键字段可缺失，缺失时 App 会自动调用 API 补全)*

### 2. 词根库 (`roots.json`) 标准结构示例：
```json
[
  {
    "root": "spect",
    "type": "词根",
    "meaning": "看 (to look)",
    "derivatives": [
      { "word": "inspect", "meaning": "检查 (向内看)" },
      { "word": "spectator", "meaning": "观众" }
    ]
  }
]
```

## 四、最佳实践总结

1. **你的日常工作流**：在电脑上用 VSCode 编辑和迭代 JSON 格式的词库 -> Commit 到 GitHub -> 用手机打开 AiWord -> 设置页点击 URL 更新 -> 瞬间加载最新库。
2. **切勿手动修改 `.bin`**：BIN 文件经过了特殊的序列化加密压缩。不要尝试用文本编辑器去打开或者维护 `.bin` 文件。数据维护永远只针对 `JSON`，让 App 去负责做“从 JSON 编译到 BIN”的苦力活。
